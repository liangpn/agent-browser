# Agent-Browser

## 基础风格

一张适合小红书发布的技术类信息图 3:4 海报，适配小红书内容流。
整体为 3D 黏土风 / 浮雕风视觉风格，
所有文字、图形与信息模块呈现为软陶或黏土雕塑质感，
具有轻微浮雕高度差与真实材质细节，边缘圆润、亲和感强。

背景为低饱和米色或浅灰色，
主色调为暖黄色、浅棕色、灰蓝色、少量科技橙点缀，
整体明亮但不刺眼，干净高级，符合小红书审美。


## 海报结构（分为两个层次）

海报从上到下分为两个视觉区域，上层更突出、更显眼：

---

### 🌟 上层区域：AI 介入（更显眼、更突出）

此区域用**更鲜艳的配色**或**发光效果**来突出，代表用户通过自然语言驱动 AI Agent 的完整闭环。

**模块1：用户输入**
- 一个黏土质感的人物头像或对话气泡
- 气泡内显示自然语言指令（如"帮我查订单123"）
- 代表用户用自然语言下达任务

**模块2：AI Agent**
- 一个黏土质感的机器人大脑或 AI 图标，**带有发光或高亮效果**
- 代表 AI Agent（如 GPT、Claude）理解用户意图并规划步骤
- 旁边可标注 **"Agent"**

**模块3：bash / shell tool**
- 一个黏土质感的终端窗口或工具箱图标
- 代表 Agent 调用 bash/shell 工具来执行命令
- 可标注 **"bash tool"** 或 **"shell"**
- 用箭头连接到下层

---

### 下层区域：Agent-Browser 原生机制

此区域为**基础色调**，展示工具本身的工作原理（不涉及 AI，纯 CLI 控制浏览器）。

**模块4：CLI 命令**
- 黏土质感的命令行终端图标
- 显示命令文字（如 `agent-browser open`, `click @e1`, `snapshot`）
- 代表通过 CLI 发送指令

**模块5：技术栈连接层**
- 黏土质感的管道或连接线
- 管道上可标注 **Rust**、**Node.js**、**Playwright**
- 另一条支线可标注 **CDP**（连接外部浏览器）

**模块6：浏览器执行层**
- 一个黏土质感的浏览器窗口图标（Chromium 风格）
- 窗口内显示简化的网页结构
- **关键**：旁边有一只"眼睛"图标，代表用户可以直接看到浏览器的操作（headed 模式或 CDP 连接本机浏览器）

---

### 底部总结区

简洁的黏土质感文字块，展示完整闭环：
"👤 自然语言 → 🤖 AI Agent → 📦 bash tool → 💻 Agent-Browser CLI → 🌐 浏览器 → 👁️ 用户可见"
